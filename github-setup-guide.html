📋 Setup Guide
How to Host on
GitHub Pages.

A free, permanent URL for all your ELEVATE games — no technical background needed. Estimated time: 10–15 minutes. You'll only need to do this once.

💡

What is GitHub Pages? GitHub is a free platform where developers store files. GitHub Pages is a feature that turns a folder of HTML files into a real, live website with its own URL — completely free, no credit card, no server setup. Your games will be available at a permanent link like yourusername.github.io/elevate-games that you can drop into any email or Webex message.

4
Turn on GitHub Pages

This is the step that turns your folder into a live website. In your repository, click Settings (top menu, gear icon), then scroll down to find "Pages" in the left sidebar.

github.com/[username]/elevate-games/settings/pages

📄 In left sidebar, click "Pages"
🌿 Under "Branch" — select main and click Save
🌐 GitHub Pages is now building your site!
⏱
Wait 1–3 minutes after clicking Save. GitHub needs a moment to build and deploy. Refresh the Settings → Pages screen and a green banner with your URL will appear when it's ready.

Test it: Open the URL in a private/incognito browser window to confirm everything works as a regular user would see it. Try on your phone too — the games are mobile-friendly.
6
Update your newsletter button

Now paste your GitHub Pages URL into the newsletter template. In the email HTML files, find every instance of [YOUR-GITHUB-PAGES-URL] and replace it with your actual URL.

href="[YOUR-GITHUB-PAGES-URL]/game-truths.html" href="https://brandonosborne-elevate.github.io/elevate-games/game-truths.html"
💡
Quick way: Open any HTML file in a text editor (TextEdit on Mac, Notepad on Windows), use Find & Replace (Cmd+H or Ctrl+H), paste [YOUR-GITHUB-PAGES-URL] in the Find field, and your real URL in the Replace field. Hit Replace All.

❓ Common Questions
Is this actually free? Will I get charged?
Yes, completely free. GitHub Pages is free for public repositories with no usage limits for static HTML files like these. No credit card required at any point.
Can people outside Apple access the URL?
Yes — that's how public GitHub Pages works. Anyone with the URL can view the games. Since these are internal training games (not sensitive data), that's fine. The facilitator answer key should be shared only via direct link, not posted publicly. If you need password protection, consider Notion (free tier) or a paid hosting platform instead.
What if I make a mistake uploading files?
You can delete and re-upload any file at any time. In your repo, click the file → click the trash can icon → commit the deletion → then re-upload the corrected version.
Can I use a custom domain (like elevate.apple.com)?
GitHub Pages supports custom domains, but setting one up through Apple's domain system would require IT involvement. For internal use, the username.github.io/elevate-games URL works perfectly.
The page isn't showing up after I enabled GitHub Pages — what do I do?
Wait 2–3 minutes and hard-refresh (Cmd+Shift+R or Ctrl+Shift+R). If it still doesn't work: (1) confirm your repo is Public in Settings → General. (2) Confirm you selected "main" branch in Settings → Pages. (3) Make sure a file called exactly index.html exists in the root of your repo.
How do I share the answer key without sharing it publicly?
Simply don't link to answer-key.html anywhere in the newsletter or Game Hub. The file exists on GitHub but no one will find it unless they have the direct URL. Share it privately in a Webex message to your facilitator group.
